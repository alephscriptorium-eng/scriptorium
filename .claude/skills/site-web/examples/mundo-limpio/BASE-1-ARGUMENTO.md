# BASE 1 — EL ARGUMENTO (nexo-atlas, inventado)

## 0 · Qué es

**Nexo Atlas es un catálogo FOSS de mapas de rutas peer-to-peer.** ⟨ficción
del ejemplo⟩ **Opera editores humanos y bots de índice sobre el mismo
contrato de lectura.** ⟨ficción⟩ **Los atlas locales viven en disco del
operador; solo se publica lo marcado para release.** ⟨ficción⟩

## 1 · Quiénes hacen qué

- Editor: cura entradas del catálogo.
- Operador: publica releases y cuida el portal.
- Bot de índice: propone enlaces; no escribe copy doctrinal.

## 2 · A quién

| Círculo | Qué debe quedarle claro |
| ------- | ----------------------- |
| Comunidades | «puedo clonar el atlas y montarlo en mi red» |
| Técnicos | «cada claim enlaza a un artefacto comprobable» |

## 3 · Cinco golpes

1. Qué es — catálogo FOSS de mapas de rutas.
2. Quiénes — editor / operador / bot de índice.
3. Piezas — fichas de ruta (origen, destino, canal).
4. Llévatela — tarball de release en el canal que funcione hoy.
5. Es real — portal + contratos inventados del ejemplo.

## 5 · Superficies y dosis

| Superficie | Golpes | Dosis |
| ---------- | ------ | ----- |
| Hero docs | 1, 5 | baja |
| Ficha de ruta | 3, 4 | media |
